Graph Representation

Recap

  • A graph is an ordered pair \((V, E)\)
  • Elements of \(V\) are vertices (or nodes)
  • Elements of \(E\) are edges
  • Graphs can be directed / undirected

Many ways to represent graphs:

  • Incidence matrix
  • Adjacency list
  • Adjacency matrix

Incidence

Definition: A vertex \(v\) is incident to edge \(e\) if \(v\) is an endpoint of \(e\).

  • Incidence matrix:
    • 1 row for each vertex
    • 1 column for each edge
    • entry \((i,j)\) is 1 if \(v_i\) is incident to \(e_j\)
    • if \(e_j\) is a self-loop at \(v_i\), then put a 2 instead
    • 0 otherwise
    • Each column has at most two 1's
  • Many wasted 0s

Example

\(e_1\) \(e_2\) \(e_3\)
\(v_1\) 1 0 0
\(v_2\) 1 1 1
\(v_3\) 0 1 0
\(v_4\) 0 0 1

Sum of Degrees

We already proved the following:

Theorem: The sum of the degrees of a graph equals \(2|E|\).

Let's look at this theorem in terms of incidence:

  • Represent the graph using an incidence matrix
  • \(\deg(v_i) = \sum\limits_{j=1}^n a_{i,j}\) (row sum)
  • Sum of degrees?
  • Column sum = \(2|E|\)
  • row sum = column sum

Adjacency List

  • \(v\) is adjacent to \(w\) if they share an edge
  • Adjacency list for \(v\): list of vertices adjacent to \(v\)
  • Adjacency list representation of \(G\): table of all adjacency lists

Example (Adjacency list for the same graph)

  • \(v_1\): \(v_2\)
  • \(v_2\): \(v_1\), \(v_3\), \(v_4\)
  • \(v_3\): \(v_2\)
  • \(v_4\): \(v_2\)

Adjacency Matrix

  • Rows / columns are vertices
  • Entry \((i,j)\) has a 1 iff edge between \(v_i\), \(v_j\)
\(v_1\) \(v_2\) \(v_3\) \(v_4\)
\(v_1\) 0 1 0 0
\(v_2\) 1 0 1 1
\(v_3\) 0 1 0 0
\(v_4\) 0 1 0 0

Isomorphism

In what ways are the following graphs similar? In what ways are they different?

Definition

Let \(G = (V_G, E_G)\) and \(H = (V_H, E_H)\) be two undirected graphs. Then \(G\) and \(H\) are isomorphic if there is \(f : V_G \to V_H\) such that for all \(u, v \in V_G\), the number of edges between \(u\) and \(v\) is the same as the number of edges between \(f(u)\) and \(f(v)\). \(f\) is called an isomorphism from \(G\) to \(H\).

Simple Graphs

Two simple undirected graphs (i.e. graphs containing no loops or multiple edges) \(G\) and \(H\) are isomorphic if there is a bijection \(f : V_G \to V_H\) such that \(u\) and \(v\) are adjacent (in \(G\)) if and only if \(f(u)\) and \(f(v)\) are adjacent (in \(H\)).

Are those graphs isomorphic?

Structure

  • Look for isomorphism invariants:
    • Number of vertices / edges
    • Degree sequence
    • Cycles of length \(n\)
    • Connectivity (more later)
    • Colorability
  • Isomorphic graphs have the same properties. But none of these imply isomorphism!

Concept

  • Isomorphic graphs are the same graph, structurally
  • Different representations of the same thing
  • Different names for vertices
  • Different order of vertices
  • But they represent the same, underlying concept

Möbius Ladder ML₃

The Möbius Ladder \( ML_3 \) is a fascinating graph that serves as an excellent example for understanding graph isomorphism and structural properties.

Construction

The Möbius Ladder \( ML_3 \) can be constructed in two equivalent ways:

Method 1: Cycle with Opposite Connections

  • Start with a 6-cycle (vertices \( v_0, v_1, v_2, v_3, v_4, v_5 \))
  • Add three "rung" edges connecting opposite vertices:
    • Edge between \( v_0 \) and \( v_3 \)
    • Edge between \( v_1 \) and \( v_4 \)
    • Edge between \( v_2 \) and \( v_5 \)

Method 2: Twisted Prism

  • Imagine a triangular prism (which would have 6 vertices and 9 edges)
  • Instead of connecting the two triangles directly, give one of the triangles a half-twist before connecting corresponding vertices
  • This "twist" creates the Möbius strip-like structure

Visual representation of \( ML_3 \) showing the 6-cycle with three connecting rungs

Properties

Property Value for \( ML_3 \) Explanation
Vertices 6 Denoted as \( |V| = 6 \)
Edges 9 Denoted as \( |E| = 9 \)
Regularity Cubic/3-regular Every vertex has degree 3
Chromatic Number 2 Bipartite, so 2-colorable
Planarity Non-planar Contains \( K_{3,3} \) as a minor
Connectivity 2-connected Requires removal of ≥2 vertices to disconnect
Girth 4 Length of shortest cycle is 4

Key Characteristics:

  • Vertex-transitive: The graph looks the same from any vertex
  • Bipartite: Vertices can be partitioned into two sets where all edges go between sets
  • Eulerian: All vertices have even degree, so Eulerian circuits exist
  • Non-Hamiltonian: Contains no Hamiltonian cycle

Isomorphism with \( K_{3,3} \)

A crucial observation is that \( ML_3 \) is isomorphic to the complete bipartite graph \( K_{3,3} \).

Proof of Isomorphism:

  1. Consider the vertex bipartition:
    • Set A = \(\{v_0, v_2, v_4\}\) (even indices)
    • Set B = \(\{v_1, v_3, v_5\}\) (odd indices)
  2. In \( ML_3 \), every vertex in Set A connects to every vertex in Set B
  3. This creates exactly the edge set of \( K_{3,3} \)

Explicit Isomorphism \( f: V(ML_3) \to V(K_{3,3}) \):

  • \( f(v_0) = a_1 \), \( f(v_2) = a_2 \), \( f(v_4) = a_3 \)
  • \( f(v_1) = b_1 \), \( f(v_3) = b_2 \), \( f(v_5) = b_3 \)

This isomorphism preserves all adjacency relationships, confirming the structural identity.

Visual demonstration of the isomorphism between \( ML_3 \) and \( K_{3,3} \)

Applications and Significance

The Möbius Ladder \( ML_3 \) is important in several contexts:

  • Graph Minor Theory: Serves as a fundamental example in Kuratowski's theorem for non-planar graphs
  • Chemical Graph Theory: Represents certain molecular structures
  • Topological Graph Theory: Demonstrates the concept of graph embeddings on non-planar surfaces
  • Computer Science: Used in network design and parallel computing architectures

The adjacency matrix should be a 6×6 matrix with a block structure corresponding to the bipartition. Each vertex in Set A should be adjacent to all vertices in Set B and vice versa, with no edges within each set.

Circular Ladder CL₃

The Circular Ladder Graph \( CL_3 \), also known as the triangular prism graph, provides a fascinating contrast to the Möbius Ladder \( ML_3 \) and serves as an excellent example for understanding graph isomorphism.

Construction

The Circular Ladder \( CL_3 \) can be constructed in several equivalent ways:

Method 1: Two Triangles with Connecting Edges

  • Start with two disjoint triangles (3-cycles)
  • Add edges connecting each vertex of one triangle to the corresponding vertex of the other triangle
  • This creates a triangular prism structure

Method 2: Cycle with Parallel Connections

  • Outer cycle: \( v_0 - v_1 - v_2 - v_0 \)
  • Inner cycle: \( u_0 - u_1 - u_2 - u_0 \)
  • Rung edges: \( v_0 - u_0, v_1 - u_1, v_2 - u_2 \)

Method 3: Cartesian Product

  • \( CL_3 \cong C_3 \times K_2 \)
  • Where \( C_3 \) is a 3-cycle and \( K_2 \) is a single edge
  • This product construction explains the "ladder" terminology

Visual representation of \( CL_3 \) showing the triangular prism structure

Properties

Property Value for \( CL_3 \) Explanation
Vertices 6 Denoted as \( |V| = 6 \)
Edges 9 Denoted as \( |E| = 9 \)
Regularity Cubic/3-regular Every vertex has degree 3
Chromatic Number 3 Not bipartite; contains triangles
Planarity Planar Can be drawn as a triangular prism
Bipartite No Contains odd cycles (triangles)
Connectivity 3-connected Highly connected structure
Girth 3 Contains 3-cycles (triangles)

Key Characteristics:

  • Vertex-transitive: The graph looks the same from any vertex
  • Hamiltonian: Contains Hamiltonian cycles
  • Eulerian: All vertices have even degree, so Eulerian circuits exist
  • Planar triangulation: Represents a polyhedron (triangular prism)

CL₃ vs ML₃: A Perfect Isomorphism Case Study

\( CL_3 \) and \( ML_3 \) form a perfect pair for studying graph isomorphism because they share many numerical properties but are structurally distinct.

Property \( CL_3 \) (Circular Ladder) \( ML_3 \) (Möbius Ladder) Same/Different
Vertices 6 6 Same
Edges 9 9 Same
Degree Sequence (3,3,3,3,3,3) (3,3,3,3,3,3) Same
Planarity Planar Non-planar Different
Bipartite No Yes Different
Chromatic Number 3 2 Different
Girth 3 (triangles) 4 (no triangles) Different
Isomorphic to K₃,₃ No Yes Different

Key Insight: This comparison demonstrates that while vertices, edges, and degree sequence are necessary conditions for isomorphism, they are not sufficient. \( CL_3 \) and \( ML_3 \) are isomorphism impostors - they appear identical numerically but have fundamentally different connectivity structures.

Non-Isomorphism with K₃,₃

Unlike \( ML_3 \), the graph \( CL_3 \) is not isomorphic to \( K_{3,3} \). Here's why:

Conclusive Proofs of Non-Isomorphism:

  1. Bipartiteness Difference:
    • \( K_{3,3} \) is bipartite (can be 2-colored)
    • \( CL_3 \) is NOT bipartite (contains triangles, requires 3 colors)
  2. Cycle Structure Difference:
    • \( K_{3,3} \) has girth 4 (shortest cycle has 4 edges)
    • \( CL_3 \) has girth 3 (contains 3-cycles/triangles)
  3. Planarity Difference:
    • \( K_{3,3} \) is non-planar (Kuratowski's theorem)
    • \( CL_3 \) is planar (can be drawn as a prism)

Isomorphism Invariants Used:

  • Bipartiteness - preserved under isomorphism
  • Girth - preserved under isomorphism
  • Planarity - preserved under isomorphism
  • Chromatic Number - preserved under isomorphism

Since these invariants differ between \( CL_3 \) and \( K_{3,3} \), the graphs cannot be isomorphic.
